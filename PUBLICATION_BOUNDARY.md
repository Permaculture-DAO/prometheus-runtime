# Publication Boundary

This repository is publicly visible to support inspection and bounded technical
review of the development runtime. It is source-available, not open source.

Public visibility does not:

- make the repository open source while its code is proprietary (`UNLICENSED`);
- create production, security, performance, empirical, legal, market, or financial
  certification;
- authorize real-world evidence ingestion or capital-facing use;
- convert synthetic or TEST data into ecological proof or admissible value;
- alter the authority of the signed canon in `prometheus-canon`.

Status is `development runtime; production admission gated`.

## Two-tier posture — proof vs power

Prometheus separates the **proof** from the **power**:

- **Verification / methodology tier** (canon prose, MRV protocol, pre-registration,
  falsifiability, data contracts) is openable under **CAL-1.0** — sharing the method
  strengthens independent scrutiny.
- **Value / claim tier** (gated ingestion, normalization, evidence packaging, and any
  logic that produces value or claims) remains **proprietary and gate-bound**, so it
  cannot be run to assert value outside the canonical gates and legal wrapper.

Any future opening of a tier is a deliberate, gate-tied, legally-reviewed decision —
never a default of public visibility. See [`LICENSE.md`](LICENSE.md).
